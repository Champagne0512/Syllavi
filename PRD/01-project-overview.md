# 01 - 项目概述

## 1.1 项目背景

### 市场痛点

中国大学生在学业管理中面临以下问题：

1. **课程表录入繁琐**
   - 每学期手动输入20+门课程
   - 教务系统导出格式不统一
   - 调课、换教室需要手动更新

2. **待办管理分散**
   - 作业deadline分布在微信群、QQ群、教务系统
   - 缺乏统一的提醒机制
   - 容易遗漏重要任务

3. **资料管理混乱**
   - 课件、笔记散落在各处
   - 考前复习找不到重点
   - 文件命名不规范难以检索

4. **专注效率低下**
   - 学习时频繁被打断
   - 缺乏专注时长统计
   - 无法形成良好学习习惯

### 解决方案

**Syllaby** 通过 AI 技术解决上述痛点：

- **拍照识别** → 自动生成课程表，减少手动输入
- **统一待办** → 集中管理所有作业和考试
- **智能摘要** → AI自动提取资料重点
- **专注追踪** → 番茄钟+数据统计，养成习惯

### 产品slogan

## 1.3 目标用户

### 用户画像

**主要用户**: 中国在校大学生

| 属性 | 描述 |
|------|------|
| 年龄 | 18-24岁 |
| 身份 | 本科生/研究生 |
| 特征 | 追求效率、重视颜值、习惯微信生态 |
| 设备 | 智能手机（iOS/Android） |

### 用户场景

#### 场景1: 新学期开学
> 小明拿到新学期课表，打开Syllaby拍照，AI自动识别所有课程信息，确认后一键导入，整个过程不到1分钟。

#### 场景2: 收到作业通知
> 老师在群里发布作业deadline，小明打开Syllaby截图识别，自动创建待办事项并设置提醒。

#### 场景3: 期末复习
> 小红有一份100页的PDF课件，她用Syllaby的"AI划重点"功能，获得5页精华摘要，大大提高复习效率。

#### 场景4: 图书馆自习
> 小张在图书馆开启专注模式，完成3个番茄钟，获得专注卡片分享到朋友圈，记录学习成就。

### 用户需求优先级

| 优先级 | 需求 | 频率 | 重要性 |
|--------|------|------|--------|
| P0 | 查看今日课程 | 每日多次 | 核心功能 |
| P0 | 管理待办任务 | 每日 | 核心功能 |
| P1 | AI识别课程表 | 每学期1-2次 | 关键卖点 |
| P1 | 专注计时 | 每周多次 | 高频使用 |
| P2 | 资料管理 | 每周 | 中频使用 |
| P2 | AI划重点 | 期末 | 关键卖点 |

## 1.5 设计理念

### Academic Zen 学术禅意

受日本"禅意设计"和苹果"简约主义"启发，Syllaby追求：

1. **Less is More**
   - 界面元素精简
   - 功能聚焦核心
   - 信息层级清晰

2. **Calm Technology**
   - 不打扰用户
   - 减少视觉噪音
   - 营造专注氛围

3. **Emotional Design**
   - 触觉反馈增强交互
   - 动画过渡自然流畅
   - 成就系统激励用户

## 1.6 技术选型

### 前端技术栈

| 技术 | 选择 | 理由 |
|------|------|------|
| 框架 | 微信小程序原生 | 生态成熟、用户基数大 |
| 样式 | WXSS + CSS变量 | 原生支持、性能好 |
| 状态 | globalData | 轻量级、够用 |

### 后端技术栈

| 技术 | 选择 | 理由 |
|------|------|------|
| BaaS | Supabase | 开源、PostgreSQL、实时订阅 |
| 认证 | Supabase Auth | 支持JWT、集成微信登录 |
| 存储 | Supabase Storage | S3兼容、CDN加速 |
| 函数 | Edge Functions | Deno运行时、全球边缘部署 |

### AI服务

| 功能 | 服务商 | 理由 |
|------|--------|------|
| OCR | 腾讯云OCR | 国内访问快、中文识别准 |
| LLM | DeepSeek/Minimax | 性价比高、中文优化 |

### 第三方服务

| 服务 | 用途 |
|------|------|
| 微信订阅消息 | 课程提醒、deadline提醒 |
| 微信云开发 | 备选后端方案 |

## 1.7 项目约束

### 技术约束

1. **小程序限制**
   - 包大小限制2MB（分包可扩展）
   - 不支持npm直接引用（需构建）
   - WebView有域名白名单限制

2. **Supabase限制**
   - 免费版：500MB数据库、1GB存储
   - Edge Functions冷启动延迟

3. **AI服务限制**
   - OCR按次计费
   - LLM Token按量计费

### 业务约束

1. **隐私合规**
   - 需获取用户授权
   - 数据存储需符合法规
   - 不收集敏感信息

2. **内容安全**
   - 上传内容需审核
   - 禁止违规内容

### 时间约束

- MVP版本：4周内完成
- 完整版本：8周内完成
